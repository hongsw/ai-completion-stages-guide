# 인공지능 완성형 활용단계 소개 가이드

> 도구 사용을 넘어, 에이전트 문명으로 가는 10단계

이 저장소는 일반인도 빠르게 따라오면서,
**"지금 내가 겪는 작은 불편 하나를 해결하는 다음 단계가 무엇인지"** 체감하도록 설계된 공개 가이드입니다.

- 메인 언어: 한국어
- 보조 언어: English
- 다국어 확장: 인구/사용자 규모 기준 상위권 언어 10종

## 왜 이 리포가 필요한가

많은 사람이 AI를 "쓴다"고 말하지만 실제 수준은 크게 다릅니다.

- 어떤 사람은 챗봇에 질문하는 수준이고
- 어떤 사람은 n8n으로 여러 서비스를 연결하며
- 어떤 사람은 멀티 에이전트 팀과 하네스를 설계하고
- 어떤 사람은 sLLM/ML을 서비스에 붙이고
- 어떤 사람은 벤치마크와 논문까지 공개합니다

이 리포는 그 차이를 **단계별로 보여주고**, 각 단계에서 **직접 해볼 수 있는 작은 실험**과 **다음 단계로 가는 불편 해소 포인트**를 제시합니다.

## 10단계 개요

1. 챗봇 수준
2. 다중 챗봇 연결 / n8n 자동화 수준
3. 스킬·플러그인·MCP 수준
4. 서브 에이전트 팀 수준
5. 에이전트 하네스 수준
6. 에이전트 오로보로스(자기개선 루프) 수준
7. 오픈소스 공개 수준
8. Generate as a Service 수준
9. sLLM / ML 직접 구현 및 서비스 적용 수준
10. Full LLM / 벤치마크 / 논문 / 서비스 통합 수준

이 문서는 단순 툴 모음이 아닙니다.

- 연구자처럼 근거를 붙이고
- 철학자처럼 단계의 의미를 묻고
- 실험가처럼 바로 시험할 수 있게 만들었습니다

핵심 질문은 하나입니다.

> AI를 "사용하는 사람"에서
> AI 시스템을 설계하고 사업화하는 사람으로
> 어떻게 이동할 것인가?

## 빠른 시작

### Level 1 — 챗봇 수준
- 불편: 반복적으로 같은 질문을 다시 해야 한다
- 체험: ChatGPT/Claude/Gemini에게 같은 업무를 3번 시켜본다
- 깨달음: "대화는 되는데, 내 흐름에 연결되진 않네"
- 다음 단계 신호: 자동화가 필요하다

### Level 2 — 자동화 수준
- 불편: 내가 붙여넣기하고 전송하는 일을 계속 하고 있다
- 체험: 폼 입력 → 요약 → 메신저 전송 자동화 1개 만들기
- 다음 단계 신호: 단순 연결 말고, 툴을 스스로 쓰는 AI가 필요하다

### Level 3 — MCP/툴 수준
- 불편: AI가 말은 하는데 실제 액션은 못 한다
- 체험: 파일 읽기/웹 검색/문서 생성 가능한 AI 환경 써보기
- 다음 단계 신호: 한 명짜리 만능 비서보다 역할 분리가 필요하다
