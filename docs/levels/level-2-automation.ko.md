# Level 2. 자동화 입문

## 이 단계의 정의
AI를 반복 업무 흐름 안에 넣어 자동으로 실행되게 만드는 단계입니다.

## 이 단계에서 얻는 능력
- 입력 → 처리 → 전달 자동화
- 반복 업무 절감
- 여러 SaaS와 AI 연결

## 대표 불편
- 예외 처리에 약함
- 규칙 기반 연결이 많음
- AI가 직접 도구를 쓴다기보다 파이프라인에 끼워진 느낌

## 왜 다음 단계가 필요한가
단순 연결만으로는 복잡한 일을 처리하기 어렵습니다.
AI가 파일, 웹, API 등 외부 도구를 직접 다뤄야 합니다.

## 1시간 실습 미션
### 미션
템플릿 기반 자동화 하나를 골라 실제로 따라 해보세요.

### 추천 흐름
- 폼 입력 → 요약 → 메신저 전달
- 이메일 → 분류 → 노션 기록
- 문서 내용 → 요약 → 슬랙/디스코드 전달

### 실습 절차
1. `awesome-n8n-templates`에서 가장 단순한 템플릿 1개를 고릅니다.
2. 입력 이벤트와 출력 채널을 파악합니다.
3. 실제로 연결했을 때 사람 손이 어디서 덜 들어가는지 관찰합니다.
4. 예외 상황이 오면 어디서 깨지는지도 적습니다.

## 실습 후 질문
- 무엇이 자동화돼서 가장 시원했나?
- 어디서 규칙이 너무 딱딱하다고 느꼈나?
- 왜 “AI가 직접 도구를 쓰는 구조”가 필요해 보였나?
