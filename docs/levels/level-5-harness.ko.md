# Level 5. 에이전트 하네스

## 이 단계의 정의
에이전트 팀을 통제 가능한 시스템으로 운영하는 단계입니다.

## 이 단계에서 얻는 능력
- 역할별 실행 통제
- 상태 관리
- 로그와 추적
- 재시도 설계
- 사람 승인 지점 설계

## 대표 불편
- 구현 난도가 올라감
- 단순 자동화보다 설계력이 필요함
- 평가 기준이 없으면 복잡성만 커질 수 있음

## 왜 중요한가
여기서부터는 “에이전트를 쓴다”가 아니라,
**에이전트 운영체계를 설계한다**는 차원으로 올라갑니다.
창업자 관점에서는 이 단계가 매우 중요합니다.

## 추천 공개 자료
- harness-100: <https://github.com/revfactory/harness-100>
- LangGraph: <https://github.com/langchain-ai/langgraph>
- mcp-agent: <https://github.com/lastmile-ai/mcp-agent>

## 1시간 실습 미션
### 미션
Level 4에서 만든 역할 구조를 “운영 가능한 구조”로 바꿔 보세요.

### 실습 절차
1. Level 4에서 만든 조사자/작성자/검토자 구조를 다시 꺼냅니다.
2. 각 단계에 대해 아래를 적습니다.
   - 입력
   - 출력
   - 실패 조건
   - 재시도 여부
   - 사람 승인 필요 여부
3. 로그가 없으면 무엇을 알 수 없는지도 적습니다.
4. harness-100 또는 LangGraph를 보며 내 구조와 비교합니다.

## 실습 후 질문
- 팀과 운영체계의 차이는 무엇인가?
- 실패했을 때 어디서 깨졌는지 추적 가능한가?
- 이 구조를 다른 프로젝트에도 재사용할 수 있는가?

## 핵심 메시지
창업자 수준이면, 멀티에이전트를 쓰는 것에서 멈추지 않고
**하네스로 운영할 줄 알아야 체급이 달라집니다.**
